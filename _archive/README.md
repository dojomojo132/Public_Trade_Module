## 🔧 PowerShell утилиты (эмуляция сканера)

| Файл | Описание |
|------|----------|
| `ComToClipboard.ps1` | Драйвер для чтения данных из COM-порта в буфер обмена |
| `ComToKeys.ps1` | Драйвер для эмуляции нажатий клавиш из COM-порта |
| `Run.bat` | Батник для запуска ComToClipboard.ps1 |

**Примечание:** Эти утилиты использовались для тестирования сканера штрихкодов через виртуальный COM-порт, но заменены на встроенный драйвер 1С и внешний эмулятор в папке `ScannerEmulator/`.

## ℹ️ Что осталось в корне проекта

**Актуальная документация (обязательна):**
- `ТЕХНИЧЕСКАЯ СПЕЦИФИКАЦИЯ КОНФИГУРАЦИИ PTM (Public Trade Module).xml` — единственный источник истины
- `Алгоритм разработки.xml` — процесс разработки
- `Стандарт_описания_объектов.xml` — шаблон документирования
- `Регламент актуализации документации.xml` — правила обновления документации
- `АУДИТ_8.3.27.md` — результаты аудита платформы
- `МИГРАЦИЯ_8.3.27.md` — руководство по миграции

**Папки конфигурации 1С:**
- `AccumulationRegisters/`, `Catalogs/`, `CommonModules/`, `Documents/`, `Enums/`, и т.д.

**Служебные папки:**
- `.github/` — инструкции для GitHub Copilot
- `.vscode/` — настройки VS Code и MCP
- `docs/` — актуальная техническая документация (стандарты 1С, XML, элементы форм)
- `mcp-config/` — конфигурация MCP-сервера для метаданных
- `ScannerEmulator/` — эмулятор сканера штрихкодов
- `Тесты/` — внешние обработки для тестирования

**Файлы конфигурации:**
- `Configuration.xml` — корневой файл конфигурации
- `ConfigDumpInfo.xml` — метаинформация выгрузки
- `package.json` — зависимости Node.js для MCP-инструментов
