Описание функционала отчета

********************************************************************

1. Открытие смены.

- Заполнение критической информации для работы отчета ( Владелец отчета, Дата смены, Ссылка на предыдущий отчет, текущая оплата закупщика).

- загрузка данных из шаблона (строки таблиц, которые повторяются каждый день)

- загрузка долговых ПН

- загрузка задолжностей закупщика

- загрузка задолжностей по ЗП

- загрузка задолжностей по рассчетам пива.

- обновление формы документа

- обновление расчетов на форме

********************************************************************

2. При изминении данных в отчете

-расчет текущего баланса (обновление оплаты закупщика, возврат текущего баланса)

- обновление данных долгу закупщика

- автосохранение отчета.

*********************************************************************

3. Закрытие смены

- формирование файла отчета

(перенос ПН в долговые, перенос неоплаченой зп,Перенос Задолжностей По Пиву, ВыгрузитьОтчетНаСервере)

-отправка отчета в телеграм

-отправка перебора

**********************************************************************

4. Показать отчет

-по очереди вызываются функции расчета и формаирования макета.

-каждая функция отчечает за конкретный блок расчета и его отображении в отчете

- т.к. у меня есть отдельная функция отвечающая за расчет, мне не нравится что тут повторный расчет. оставить в функциях только вывод макета и подсчет сумм для блоков макета.
